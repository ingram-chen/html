📖 BMAD-METHOD 是什麼？

BMAD-METHOD 是一個革命性的 AI 驅動敏捷開發框架，專為軟體和遊戲開發設計。它不只是給你答案，而是透過引導式反思幫助你發現更好的解決方案。

🎯 核心理念：BMAD-CORE

BMAD 代表四個核心支柱：

  • Brilliance（卓越）：透過結構化流程將 AI 推向極致表現
  • Mastery（精通）：專業化代理和領域專業知識
  • Adaptability（適應性）：跨任何規模自動擴展，從錯誤修復到企業級系統
  • Depth（深度）：保持完整上下文的敘事性推理

加上兩個賦能元素：

  • Reflection（反思）：策略性提問解鎖突破性解決方案
  • Engine（引擎）：協調 19+ 個專業代理和 50+ 個工作流程的框架

🌟 為什麼選擇 BMAD-METHOD？

🎯 規模自適應智能

自動根據專案需求調整規劃深度和文檔，支援從單一錯誤修復到企業級系統的各種規模。

🤖 專業代理團隊

19+ 個專業 AI 代理，包括 PM、架構師、開發者、測試工程師、UX 設計師等，協同工作。

📋 結構化工作流程

50+ 個預建工作流程，涵蓋從需求分析到實作部署的完整開發週期。

🎨 可客製化框架

完整的客製化支援，可建立自己的代理、工作流程和模組。

💡 創意思維工具

內建創意智能套件，提供腦力激盪、設計思維、問題解決等創新方法。

🔄 更新安全設計

客製化設定獨立儲存，框架更新不會影響你的個人化配置。

💼 主要應用場景

  • 軟體開發專案：從小型功能到大型平台的完整開發流程
  • 遊戲開發：專門的遊戲設計師、架構師和開發者代理支援
  • 產品規劃：完整的 PRD（產品需求文件）和技術規格制定
  • 既有專案改進：為現有程式碼庫添加結構化開發流程
  • 創意發想：使用創意智能套件進行創新和問題解決
  • 企業級開發：包含安全性、DevOps 和測試策略的擴展規劃

✨ 核心特色

🎯 三軌規劃系統

BMAD-METHOD 會根據專案需求自動調整規劃深度：

⚡ Track 1 Quick Flow

快速實作軌道

  • 僅需技術規格文件
  • 適用於：錯誤修復、小功能、明確範圍的任務
  • 快速部署，幾分鐘內完成

📋 Track 2 BMad Method

完整規劃軌道

  • PRD + 架構 + UX 設計
  • 適用於：產品、平台、複雜功能
  • 全面的文檔和規劃支援

🏢 Track 3 Enterprise Method

企業級規劃軌道

  • BMad Method + 安全性/DevOps/測試
  • 適用於：企業需求、合規性要求
  • 最完整的規劃和控制

🏗️ 四階段開發方法論

  1. 階段 1：分析（Analysis） - 選擇性 - 腦力激盪、研究、產品簡報
  2. 階段 2：規劃（Planning） - 必要 - 規模自適應的 PRD/技術規格/GDD
  3. 階段 3：解決方案（Solutioning） - 視軌道而定 - 架構、安全、DevOps、測試策略
  4. 階段 4：實作（Implementation） - 迭代式 - 以用戶故事為中心的開發，及時提供上下文

  • Game Designer（遊戲設計師）：遊戲機制和玩法設計

🧩 三大核心模組

📋 模組 1：BMad Method (BMM) - AI 驅動敏捷開發

BMad Method 是革命性的 AI 驅動敏捷框架，專為軟體和遊戲開發設計。自動從單一錯誤修復調整到企業級系統。

v6 版本重點特色：

  • 規模自適應智能：三軌規劃系統自動調整
  • 四階段方法論：從分析到實作的完整流程
  • 12 個專業代理：涵蓋所有開發角色
  • 34 個工作流程：完整的階段性參考指南

主要工作流程範例：

  • *workflow-init - 初始化專案工作流程路徑
  • *quick-spec-flow - 快速規格流程（錯誤修復和小功能）
  • *prd - 產品需求文件制定
  • *architecture - 系統架構設計
  • *dev-story - 用戶故事開發
  • *brainstorm-project - 專案腦力激盪
最佳實踐：不確定選擇哪個軌道？執行 *workflow-init，讓 BMM 分析你的專案目標並推薦正確的軌道。

🔧 模組 2：BMad Builder (BMB) - 建立客製化解決方案

使用 BMad-CORE 框架建立你自己的代理、工作流程和模組。

你可以建立：

  • 客製化代理：具有專業知識的領域專家
  • 引導式工作流程：任何任務的多步驟流程
  • 完整模組：特定領域的完整解決方案
  • 三種代理類型：完整模組、混合式或獨立式

適用於：

建立領域特定解決方案（法律、醫療、金融、教育、創意等）或使用客製化開發工作流程擴展 BMM。

核心工作流程：

  • create-agent - 建立客製化代理
  • create-workflow - 設計引導式流程
  • create-module - 封裝完整解決方案

💡 模組 3：Creative Intelligence Suite (CIS) - 創新與創意

使用經過驗證的方法論和技術進行 AI 驅動的創意引導。

5 個互動式工作流程：

  • Brainstorming（腦力激盪） - 使用 30+ 種技術生成和精煉想法
  • Design Thinking（設計思維） - 以人為中心的問題解決
  • Problem Solving（問題解決） - 系統化突破技術
  • Innovation Strategy（創新策略） - 顛覆性商業模式思考
  • Storytelling（說故事） - 引人入勝的敘事框架

5 個專業代理：

每個都具有獨特的引導風格和領域專業知識，協助你進行創意思考和創新。

共享資源：CIS 工作流程可被其他模組使用（例如 BMM 的 brainstorm-project 使用 CIS 腦力激盪）。

🚀 詳細安裝步驟

方法 1：使用 npx 安裝（建議）

安裝 v6 Alpha 版本（建議新專案使用）：

npx bmad-method@alpha install

安裝穩定版 v4（生產環境）：

npx bmad-method install

安裝程式提供的選項：

  1. 模組選擇 - 選擇 BMM、BMB、CIS（或全部安裝）
  2. 配置設定 - 你的名稱、語言偏好、遊戲開發選項
  3. IDE 整合 - 自動為你的 IDE 設置

安裝後的資料夾結構：

your-project/
└── {bmad_folder}/           # 預設名稱為 .bmad
    ├── core/                # 核心框架 + BMad Master 代理
    ├── bmm/                 # BMad Method (12 代理, 34 工作流程)
    ├── bmb/                 # BMad Builder (1 代理, 7 工作流程)
    ├── cis/                 # Creative Intelligence (5 代理, 5 工作流程)
    └── _cfg/                # 你的客製化設定（更新時保留）
        └── agents/          # 代理客製化檔案

方法 2：使用 Web Bundles（無需安裝）

如果你不想在本地安裝，可以下載預建的代理包，直接在 Claude Projects、ChatGPT 或 Gemini 中使用。

前往 Web Bundles 頁面

Web Bundles 在每次 main 分支提交時自動更新，始終保持最新狀態。

🔧 配置說明

基本配置：

安裝過程中會詢問以下資訊：

  • 你的名稱：用於代理互動中的個人化稱呼

🎯 快速開始指南

步驟 1：載入代理

在你的 IDE 中載入任何 BMAD 代理。各 IDE 的具體操作：

  • Claude/Claude Code：將代理檔案拖放到對話視窗
  • Cursor/Windsurf：使用 IDE 的檔案匯入功能
  • VS Code with Copilot：開啟代理檔案作為上下文

步驟 2：初始化工作流程

載入代理後，執行初始化工作流程：

*workflow-init

或使用自然語言：

Run workflow-init

這個工作流程會：

  • 分析你的專案目標
  • 推薦適合的規劃軌道
  • 設置工作流程路徑
  • 準備開發環境

步驟 3：選擇你的規劃軌道

⚡ 軌道 1：Quick Flow

適用於：

  • 🐛 錯誤修復（幾分鐘內完成）
  • ✨ 小功能（2-3 個相關變更）
  • 🚀 快速原型開發

使用方式：

*quick-spec-flow

📋 軌道 2：BMad Method

適用於：

  • 產品和平台開發
  • 完整規劃（PRD/GDD）
  • 架構決策
  • 以用戶故事為中心的實作

使用方式：

*prd
*architecture
*dev-story

🏢 軌道 3：Enterprise Method

適用於：

  • 企業級需求
  • 合規性要求
  • 安全性審查
  • DevOps 策略

使用方式：

完整 BMad Method
+ 安全性規劃
+ DevOps 策略
+ 測試策略
💡 專業建議：不確定選擇哪個軌道？執行 *workflow-init，讓系統根據你的專案目標自動推薦。

步驟 4：特殊情況 - 既有專案（Brownfield）

如果你要在現有程式碼庫中使用 BMAD-METHOD：

  1. 先記錄現有程式碼
  2. 然後選擇 Quick Flow 或 BMad Method 軌道
  3. 參考 Brownfield Guide 進行整合

🎯 執行工作流程的三種方式

方式 1：代理選單（建議初學者使用）

  1. 在 IDE 中載入代理
  2. 等待選單顯示可用的工作流程
  3. 使用自然語言或快捷方式告訴代理要執行什麼：
    • 自然語言：「Run workflow-init」
    • 快捷方式：*workflow-init
    • 選單編號：「Run option 2」

方式 2：直接斜線命令

直接使用斜線命令執行工作流程：

/bmad:bmm:workflows:workflow-init
/bmad:bmm:workflows:prd
/bmad:bmm:workflows:dev-story

優點：

  • ✅ 可混合搭配任何代理與任何工作流程
  • ✅ 執行不在載入代理選單中的工作流程
  • ✅ 熟練使用者更快速的存取方式
提示：雖然可以在不載入代理的情況下執行這些命令，但仍建議載入代理 - 對某些工作流程會有差異。

方式 3：Party Mode 執行

使用多代理協作執行工作流程：

  1. 啟動 party mode：/bmad:core:workflows:party-mode
  2. 執行任何工作流程 - 整個團隊將協同合作
  3. 獲得來自多個專業代理的不同觀點

適用於：策略決策、複雜工作流程、跨職能任務

💻 實際使用範例

範例 1：快速修復錯誤（Quick Flow Track）

場景：修復登入頁面的表單驗證錯誤

步驟：

  1. 載入 Developer 代理
  2. 執行：*quick-spec-flow
  3. 描述問題：「登入表單在空白輸入時沒有顯示錯誤訊息」
  4. 代理會：
    • 快速分析問題
    • 生成技術規格
    • 提供修復程式碼
    • 建議測試案例

預期時間：5-15 分鐘

範例 2：開發新功能（BMad Method Track）

場景：為電商平台添加購物車功能

步驟：

  1. 執行 *workflow-init 初始化專案
  2. 執行 *prd 建立產品需求文件
    • 載入 PM 代理
    • 定義功能需求、用戶故事、驗收標準
  3. 執行 *architecture 設計系統架構
    • 載入 Architect 代理
    • 設計資料模型、API 端點、前後端架構
  4. 執行 *dev-story 開始迭代開發
    • 載入 Developer 代理
    • 逐個用戶故事實作
    • 即時獲得上下文和指導

預期時間：數小時到數天（取決於功能複雜度）

範例 3：遊戲開發專案

場景：開發一款 2D 平台跳躍遊戲

步驟：

  1. 執行 *brainstorm-project 進行創意發想
    • 使用 CIS 腦力激盪工作流程
    • 探索遊戲機制、美術風格、故事情節
  2. 載入 Game Designer 代理，建立 GDD（遊戲設計文件）
    • 遊戲機制設計
    • 關卡設計
    • 角色和敵人設計
  3. 載入 Game Architect 代理，設計技術架構
    • 遊戲引擎選擇
    • 系統架構設計
    • 效能優化策略
  4. 載入 Game Developer 代理，開始實作
    • 核心遊戲循環
    • 物理系統
    • UI/UX 實作

預期時間：數週到數月（取決於遊戲規模）

範例 4：企業級專案（Enterprise Method Track）

場景：開發符合 GDPR 規範的客戶資料管理系統

步驟：

  1. 執行完整 BMad Method 流程（PRD + Architecture）
  2. 額外規劃：
    • 安全性規劃：資料加密、存取控制、稽核日誌
    • 合規性檢查：GDPR、資料隱私權、使用者同意機制
    • DevOps 策略：CI/CD、自動化測試、部署流程
    • 測試策略：單元測試、整合測試、安全測試
  3. 使用 Party Mode 進行關鍵決策
    • 多代理協作評估安全性設計
    • 確保各方面考量周全

預期時間：數月（包含完整的企業級規劃和合規性檢查）

範例 5：建立客製化代理（BMB 模組）

場景：為法律科技新創公司建立法律顧問代理

步驟：

  1. 載入 BMad Builder 代理
  2. 執行：create-agent 工作流程
  3. 定義代理特性：
    • 名稱：Legal Advisor
    • 專業領域：合約審查、法律諮詢、合規性檢查
    • 知識庫：法律條文、判例、最佳實踐
  4. 執行：create-workflow 建立專屬工作流程
    • 合約審查流程
    • 風險評估流程
    • 法律建議生成流程
  5. 執行：create-module 封裝為完整模組
    • 整合所有代理和工作流程
    • 建立模組文檔
    • 準備分享或內部使用

預期時間：數小時到數天（取決於複雜度）

範例 6：創意工作坊（CIS 模組）

場景：產品創新腦力激盪會議

步驟：

  1. 載入 CIS 創意代理（選擇適合的風格）
  2. 執行：brainstorming 工作流程
    • 使用 30+ 種腦力激盪技術
    • 生成大量創新想法
    • 精煉和評估想法
  3. 執行：design-thinking 工作流程
    • 同理心階段：理解用戶需求
    • 定義階段：界定問題
    • 發想階段：產生解決方案
    • 原型階段：快速原型設計
    • 測試階段：驗證想法
  4. 執行：innovation-strategy 制定創新策略
    • 商業模式創新
    • 市場定位策略
    • 顛覆性創新機會

預期時間：1-4 小時（取決於工作坊深度）

❓ 常見問題排解

安裝相關

Q1: 安裝時出現 Node.js 版本錯誤

問題：錯誤訊息顯示「Node.js version is too old」

解決方案：

  • 確認 Node.js 版本：node --version
  • 需要 v20 或更高版本
  • 前往 nodejs.org 下載最新版本
  • 或使用 nvm：nvm install 20 然後 nvm use 20

Q2: npx 命令無法執行

問題：終端機顯示「npx: command not found」

解決方案：

  • npx 應該隨 Node.js 一起安裝
  • 重新安裝 Node.js
  • 檢查環境變數 PATH 設定
  • 嘗試使用 npm：npm install -g bmad-method@alpha 然後 bmad-method install

Q3: 找不到 {bmad_folder} 資料夾

問題：安裝後看不到 bmad 相關資料夾

解決方案：

  • 預設資料夾名稱為 .bmad（注意開頭的點）
  • 在 Mac/Linux 上，以點開頭的資料夾是隱藏的
  • 使用 ls -la 查看隱藏資料夾
  • 在檔案管理器中啟用「顯示隱藏檔案」選項

使用相關

Q4: 不確定選擇哪個規劃軌道

解決方案：

  • 執行 *workflow-init
  • 描述你的專案目標
  • 系統會自動分析並推薦適合的軌道

快速參考：

  • 錯誤修復、小改動 → Quick Flow
  • 新產品、大功能 → BMad Method
  • 企業專案、需合規 → Enterprise Method

Q5: 代理選單沒有顯示

問題：載入代理後沒有看到工作流程選單

解決方案：

  • 等待幾秒鐘，AI 需要時間載入和處理
  • 檢查是否正確載入了代理檔案
  • 嘗試重新載入代理
  • 使用直接斜線命令代替：/bmad:bmm:workflows:workflow-init

Q6: 工作流程執行失敗

問題：執行工作流程時出現錯誤

解決方案：

  • 檢查錯誤訊息，了解具體問題
  • 確認代理檔案完整且未損壞
  • 嘗試重新執行工作流程
  • 檢查專案結構是否正確
  • 查看 GitHub Issues 看是否有類似問題

Q7: 如何更新 BMAD-METHOD？

更新步驟：

  • 執行相同的安裝命令：npx bmad-method@alpha install
  • 系統會檢測並更新框架
  • 你的客製化設定會被保留（儲存在 _cfg/ 中）
  • 更新完成後，重新載入代理即可使用新版本

進階問題

Q8: 如何啟用文件分片以節省 token？

文件分片可以為大型專案節省 90%+ 的 token 使用量。

使用方式：

  • 系統提供內建工具可自動分割文件
  • 按標題分割文件
  • 工作流程會自動偵測並處理分片文件
  • 查看 Document Sharding Guide 了解詳細設定

Q9: 可以自訂代理的行為嗎？

可以！BMAD-METHOD 提供完整的客製化支援。

客製化方式：

  • 修改 _cfg/agents/ 中的代理設定檔
  • 自訂代理名稱、角色、行為
  • 調整工作流程參數
  • 建立全新的代理和工作流程（使用 BMB 模組）

參考 Agent Customization Guide 了解詳細說明。

Q10: 如何在團隊中分享客製化設定？

分享方式：

  • 將 _cfg/ 資料夾加入版本控制
  • 團隊成員 clone 專案後即可獲得相同設定
  • 或使用 Web Bundles 建立可分享的代理包
  • 參考 Web Bundles Guide 了解如何建立和分享
